# Контрагенты

Published 2023-05-19

Справочник содержит информацию о клиентах и поставщиках для дальнейшего использования в процессе оформления документов.

Справочник находится Главное -> Контрагенты , так же его можно найти Справочники -> Работа с контрагентами -> Контрагенты .

Справочник имеет иерархическую структуру, контрагентов можно объединять по группам и подгруппам. Справочник имеет 2 формы: основную или упрощенную.

На упрощенной форме не отображаются долги и переплаты КА.

Как включить упрощенную форму описано в разделе Персональные настройки.

# Дополнительные реквизиты выведенные на форме списка:

Ответственный - закреплённый за контрагентом менеджер на текущий момент времени.

Долг - отображается общий остаток взаиморасчетов по контрагенту на текущую дату, в разрезе всех имеющихся договоров.

# Команды:

служит для совершения звонка клиенту. Функционал работает только при включенной МИКО панели телефонии.

служит для формирования заказа покупателя.

служит для создания контакта оператора.

служит для создания документов Реализация товаров и услуг, Возврат товаров от покупателя.

служит для создания документов Акт передачи оборудования и Акт приема оборудования, Оформление аренды.

На форме есть фильтр по статусу. По умолчанию установлен статус Активный , система отображает активные адреса, у которых есть доставки.

На форме есть фильтр по периоду, адресу доставки, по статусу заказа. На основании заказа покупателя в статусе Выполнен можно создать новый заказ при помощи кнопки Повторить заказ . Данные из выполненного будут перенесены в новый заказ.

служит для формирования отчета Список документов по контрагентам.

# Фильтры:

В форме списка у КА отображаются переплаты. Сумма переплаты отображается в колонке Долг со знаком минус. При включении фильтра у КА не отображаются переплаты.

По умолчанию не заполнен. При указании даты в форме списка у КА отображается наличие долга с указанной даты и позднее. Сумма долга отображается в колонке Долг с наличием долга выделены красным цветом.

Включает отображение журнала работы с клиентом, на форме в нижней части справочника отображаются записи из журнала работы с клиентом.

# Форма элемента

Наименование - следует вводить краткое наименование КА, по которому будет осуществляться поиск в списке.

Наименование полное - следует вводить полное наименование КА, которое будет фигурировать в документах.

Код клиента - код КА, используется для поиска, заполняется автоматически.

Комментарий - служит для ввода комментария. Данный комментарий будет отображаться в заказе покупателя при оформлении отгрузки на любой адрес доставки КА, выделен оранжевым цветом и служит напоминанием оператору по данному КА.

# Основные реквизиты контрагента:

Данные реквизиты необходимо обязательно заполнить, они участвуют в выводе на печать.

К ним относятся поля: Наименование , Наименование полное , Юр./физ.лицо , ИНН , КПП , ОГРН , Код по ОКПО , Вид деятельности , Фактический адрес , Юридический адрес , Почтовый адрес , Телефон для печати , Факс , Электронная почта , Платильщик , Основной р/с .

Поскольку большинство полей, которые необходимо заполнить, вполне очевидны, остановимся только на тех, которые могут вызвать вопросы.

Телефон для печати - необходимо ввести номер телефона КА. Этот номер будет печататься в бухгалтерских документах.

Плательщик - заполняется в том случае, если заданного КА платит другой КА. При формировании документов поле будет заполняться юридическим адресом указанного плательщика. Если плательщик не выбран, то в документы будет проставляться юридический адрес данного КА.

Основной р/с – основной банковский счет контрагента, данный счет будет автоматически заполняться в платежных документах.

# Функциональные поля:

  • Оригинальные подписи – признак заполнения документов подписями руководителей. По умолчанию признак отключен и при печати документов подставляются факсимиле подписантов. Если этот признак включить, то при печати документов подписи не будут выведены.

  • При печати брать факт.адрес из адреса доставки – признак печати адреса доставки в бухгалтерских документах. По умолчанию признак отключен и при печати бухгалтерских документов подставляется фактический адрес КА. Если этот признак включить, то при формировании документов адрес будет браться из адреса доставки.

  • Пакетный ввод накладных – признак пакетной печати товарной накладной. Порядок работы с данным функционалом описан в разделе Формирование пакетных накладных.

  • Формировать,но не печатать документы - признак печати документов. Если этот признак включить, то документы будут сформированы, но на печать будут не выведены. Необходимо,если КА работает с ЭДО и ему не требуется подписание бумажных оригиналов.

# Технические реквизиты:

Код - код справочника КА .

Код бух. – код КА в бухгалтерии, используется в обработке по выгрузке данных.

Не выгружать – если установлен этот признак учета, то данный КА и документы по нему не будут переносится при обработке в бухгалтерию.

# Прочие реквизиты:

Оператор – указывается менеджер компании, закрепленный за данным КА.

Категория - указывается категория КА.

Комментарий оператора – оператор вводит свой комментарий по данному КА.

Вкладка содержит информацию о контактных лицах КА.

Порядок работы по созданию контактного лица КА описан в разделе Создание контактного лица.

Вкладка содержит информацию о договорах КА.

Порядок работы по созданию договора описан в разделе Создание договора.

Вкладка содержит информацию об адресах КА.

Порядок работы по созданию адреса доставки описан в разделе Создание адреса доставки.

Вкладка содержит информацию об оборудовании, находящемся у КА.

В столбце Дней до обслуживания указывается количество дней до планового профилактического обслуживания. Красным цветом выделяется количество дней, у которого просрочена дата профилактики с указанием количества дней просрочки.

Синим цветом выделяется оборудование, которое было выдано КА в качестве подменного на время ремонта его оборудования.

Зеленым цветом выделяется оборудование, которое находится в аренде у КА.

На форме имеются кнопки для создания документов номерного оборудовния. Порядок работы с ними описан в разделе Документы номерного оборудования.

На вкладке отображается журнал работы с КА по всем адресам доставки, а так же по КА в целом без указания адреса доставки.

Порядок работы по добавлению записи в журнал описан в разделе Занесение жалоб и обращений.

На вкладке отображается список документов, находящихся в очереди печати.

Документ можно открыть и просмотреть. По каждому документу указывается адрес доставки, и количество экземпляров.

При оформлении заказа покупателя документы по соответствующему адресу доставки подтянутся в заказ. После проведения заказа документ с данной вкладки очищается.

В данном разделе система показывает историю звонков с КА. Функционал работает при подключенной панели телефонии.

Данная вкладка доступна при включенной константе Расширенные настройки реквизитов печати бухгалтерских документов . Как включить константу описано в разделе Настройки программы.

На вкладке отображаются поля, в которые при необходимости заносятся дополнительные реквизиты.

Если эти поля заполнены, то при формировании документов реквизиты берутся из этой вкладки, в противном случае берутся основные реквизиты из карточки КА и АД.

# Права доступа

Справочник доступен всем пользователям для просмотра, создания и редактирования. Однако, нужно обратить внимание на поля Оператор и Категория , они доступны для редактирования пользователям с ролями Руководитель , Контроль качества .
