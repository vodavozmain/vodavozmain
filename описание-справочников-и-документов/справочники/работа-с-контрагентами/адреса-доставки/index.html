# Адреса доставки

Published 2023-05-19

Справочник содержит информанию об адресах, по которым осуществляется доставка воды контрагентам.

Справочник находится Главное -> Адреса доставки , так же его можно найти Справочники -> Работа с контрагентами -> Адреса доставки .

Адреса доставки создаются и редактируются из карточки Контрагента на вкладке Адреса доставки .

# Дополнительные реквизиты выведенные на форме списка:

Дата доставки - отображается последняя дата доставки на адрес.

Статус - отображается текущий на момент времени статус АД.

Менеджер - закреплённый за контрагентом менеджер на текущий момент времени.

# Командная панель

# Команды:

служит для создания контакта оператора и проверки отвала.

служит для формирования заказа покупателя.

служит для создания документов Реализация товаров и услуг, Возврат товаров от покупателя, Счет на оплату.

служит для создания документа Оформление аренды.

служит для просмотра заказов по данному КА.

служит для формирования отчета Список документов по контрагентам.

# Фильтры:

  • Маршрут - при включении и выборе маршрута на форму будут выведены адреса, которым присвоен данный маршрут.

  • Дата доставки - при включени и указании даты доставки на форму будут выведены адреса с датой последней доставки.

  • Статус - по умолчанию установлено значение Активный , при включении и выборе статуса на форму будут выведены адреса, которым присвоен данный статус.

  • Менеджер - при включении и выборе менеджера на форму будут выведены адреса, которые закреплены за данным менеджером.

  • Вид планирования - при включении и выборе вида планирования на форму будут выведены адреса, которым присвоен данный вид планирования.

# Форма элемента

Контрагент – заполняется автоматически тем КА, для которого создается АД.

Адрес доставки - необходимо заполнить при помощи адресного классификатора ФИАС.

Представление - необходимо выбрать один из вариантов представления АД предложенных системой, отредактировать предложенный вариант или ввести свой, оно печатается в МЛ и предназначено для водителя, чтобы было понятно куда доставить воду.

Статус адреса - справочно отображается статус АД. Если клиент новый и не было доставок, то статус будет установлен Новый . Если есть доставка, статус будет установлен Активный . Если был отвал, то статус будет установлен Закрыт . Если в организации есть отдел по контролю отвалов, то при отвале сначала будет установлен статус адреса Проверка , после подтверждения отвала статус будет установлен Закрыт .

Дата последней отгрузки - справочно отображается дата последней отгрузки КА. Если КА новый, то поле будет пустым.

План обзвона - заполняется автоматически последним контактом с контрагентом.

Комментарий - служит для ввода комментария. Данный комментарий будет отображаться в заказе покупателя при оформлении отгрузки на данный адрес доставки КА, выделен красным цветом и служит напоминанием оператору по данному АД.

# Описание вкладок формы элемента

Вкладка содержит основную информацию о адресе доставки.

# Основные реквизиты:

Договор - необходимо выбрать договор по которому будет работать КА на данном адресе доставки и он нужен для автоматического заполнения данных документов.

Маршрут - необходимо выбрать маршрут доставки, в дальнейшем его может изменить логист для более удобной работы в распределении заказов.

Интервал - необходимо указать интервал времени доставки по маршруту.

Дни доставки - указывается по каким типам дней клиенту можно осуществлять доставку. В дальнейшем система будет контролировать эти дни и в соответствии с ними планировать дату следующей доставки.

# Контактная информация для курьера:

График - указывается начало/окончание/перерывы работы и контрагента.

Перезвонить за (минут) - введите время за которое необходимо перезвонить клиенту, указанное время автоматически проставляется в Заказе покупателя , при необходимости время можно изменить.

Необходим грузчик - устанавливается, когда необходим грузчик, значение проставляется автоматически в Заказе покупателя , Маршрутном листе и логист видит, что необходимо выделить дополнительного сотрудника на доставку.

Примечание водителю – данное примечание печатается в МЛ для водителя или службы доставки. Например, можно указать код входной двери и т.п.

Контактное лицо - выберите ответственное лицо за постановку заказа, с ккоторым будет осуществляться связь по новым заказам и текущей доставке.

# Информация по печати документов:

Доставка документов - указывается на какой адрес осуществляется доставка документов: на текущий адрес или на другой адрес КА из списка всех его адресов. Если переключатель установлен в значение На адрес и указан адрес, то комплект документов будет добавлен в очередь печати на адрес, указанный в доставке документов и при оформлении заказа на этот адрес будут добавлены в заказ.

Комплект документов – по согласованию с КА флажками указываются те документы и их количество, которые необходимо сформировать и передать ему вместе с заказом.

Печатать клиентскую часть БЗ - устанавливается, если контагенту нужно печатать БЗ и в справочнике Организации установлена константа Не печатать клиентскую часть БЗ .

Не формировать чек – чек не будет сформирован.

Электронный чек – чек будет сформирован и отправлен по электронной почте.

Чек до отгрузки - устанавливается в том случае, если клиенту необходимо отвезти документы с текущим заказом. При этом чеки для этого адреса будут формироваться при нажатии кнопки Сформировать чеки в МЛ.

# Аналитика:

Менеджер – указыватется менеджер, который подключил данного КА к работе с организацией.

Причина подключения - указывается причина подключения, например, по рекомендации наших клиентов, узнал из интернета и т.д.

Значения данных полей вляют на статистику продаж, для дальнейшей оценки работы менеджеров.

Вкладка содержит информацию о виде планирования, по которому будет рассчитываться дата следующей доставки и формироваться задача оператору для обзвона клиентов. Доступны следующие виды планирования:

# По дням недели:

Данный вид планирования удобен при работе с юридическими лицами, когда они часто берут воду несколько раз в неделю, а также если клиент находистся в удаленном районе и доставка осуществляется в определенный день недели.

Периодичность - служит для указания периода, через который будет планироваться дата отгрузки.

Если периодичность 2 и более,то установить можно только один день недели.

Укажите в соответствующих полях по каким дням недели КА можно осуществлять доставку.

# Через каждые Х дней:

Используется если у КА есть определенный график потребления воды или с ним заранее договорились о поставке через равное количество дней.

Промежуток дней - укажите промежуток,через который КА можно осуществлять доставку, например, каждые 10 дней. Соответственно система будет планировать дату отгрузки через каждые 10 дней с учетом производственного календаря. Если такой день будет выпадать на день когда нет доставки, то система будет передвигать плановую дату отгрузки на следующий рабочий день.

# По средней:

Используетя для работы с физическими лицами, чтобы напоминать им о том, что вода заканчивается.

Начальное потребление - укажите предполагаемое потребление воды.

Задержка обзвона - добавляет заданное n-количество дней перед звонком клиенту.

Тип адреса - укажите (офис или квартира).

Среднее потребление - для офиса по умолчанию установлено 2л, для квартиры 0,5л (при необходимости значение можно изменить).

Количество человек - укажите количество человек.

После первой доставки система возьмет за норму потребления воды в день начальное потребление и спрогнозирует с учетом текущей отгрузки плановую дату следующей отгрузки.

В дальнейшем система будет производить расчет коэффициента среднего потребления воды в день и с учетом полученного коэффициента спрогнозирует плановую дату следующей отгрузки. Так же система будет учитывать типы дней при прогнозировании плановой даты следующей поставки.

# Не планировать:

Система не будет планировать дату следующей поставки,и задача оператору формироваться не будет.

  • Показывать в задачах оператора всегда – при включении данный адрес доставки всегда будет показываться в оперативных задачах.

  • Учитывать залоги при расчете тары - при расчете тары в заказе покупателя для данного адреса залог будет вычитаться из общего количества тары.

Настройки push-уведомлений рассмотрим в разделе Приложение клиента и сайты.

Вкладка содержит информацию о товарах и услугах, которая будет автоматически проставляется при оформлении заказа покупателя.

Если клиент новый и воду не заказывал, то вкладка будет пустой. Товары надо подобрать вручную при помощи кнопки Подбор .

Проведенная реализация автоматически вносит номенклатуру на вкладку.

  • Использовать ограничение на отгрузку - позволяет установить ограничение количества отгружаемой продукции в заказе покупателя. Контроль ведется по номенклатурной группе.

Количество контролируемой отгрузки - необходимо проставить количество.

В табличной части действует ограничение по количеству на номенклатурную группу.

При формировании заказа покупателя, если хоть одна номенклатурная позиция на которую установлено ограничение будет выбрана в количестве, указанном меньше ограничения, то система выдаст предупреждение.

На вкладке отображается журнал работы с КА по данному адресу доставки.

Порядок работы по добавлению записи в журнал описан в разделе Занесение жалоб и обращений.

На вкладке отображаются документы Реализация товаров и услуг за прошедший и текущий месяцы, количество бутылок в заказе и количество отгруженных бутылок по факту.

Данная вкладка доступна при включенной константе Расширенные настройки реквизитов печати бухгалтерских документов . Как включить константу описано в разделе Настройки программы.

На вкладке отображаются поля, в которые при необходимости заносятся дополнительные реквизиты.

Если эти поля заполнены, то при формировании документов реквизиты берутся из этой вкладки, в противном случае берутся основные реквизиты из карточки КА и АД.

Вкладка предназначена для настройки параметров пропуска.

Данная вкладка доступна при включенной константе Использовать пропуска . Как включить константу описано в разделе Настройки программы.

Порядок работы по данному функционалу описан в разделе Работа с пропусками.

# Права доступа

Справочник доступен всем пользователям для просмотра, создания и редактирования. Однако, нужно обратить внимание на поля Интервал , Менеджер , Причины подключения , они доступны для редактирования пользователям с ролями Руководитель и Контроль качества .
